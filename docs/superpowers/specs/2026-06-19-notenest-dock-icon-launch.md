# NoteNest — Dock Icon + Launch-to-Fresh-Note

**Date:** 2026-06-19
**Status:** Approved design, ready for implementation plan
**Builds on:** `2026-06-18-notenest-swift-redesign.md` (the shipped Swift app)

## Purpose

Improve speed of capture. Two related improvements:

1. Give NoteNest a distinct **Dock icon** so it's recognizable (today it shows
   the generic blank app icon), and use launching it as the "start writing"
   entry point.
2. On launch, land the user on a **ready-to-write note** without cluttering
   `~/Notes` with empty files.

This is the first enhancement after the initial Swift MVP shipped.

## Decisions (from brainstorming)

- **Dock, not menu bar.** No global hotkey (explicitly declined).
- **Dock-click behavior = "B":** launching the app opens a fresh/ready note;
  when the app is already running, clicking the Dock icon just brings the
  existing window forward (macOS default). New notes while open are made with
  `⌘N`.
- **Empty-note handling = "A":** on launch, if the most-recently-modified note
  is empty, reuse it (open it) instead of creating another; otherwise create a
  new note. Empty notes never pile up.
- **Icon source = generated.** A clean, minimal icon is generated by the
  project (no external design dependency) and is swappable later by replacing
  the source image and rebuilding.

## Behavior

### Launch (app was not running)
- `ensureFolderExists()`, `reload()`.
- If there are no notes → create one and open it.
- Else if the most-recent note (first in the modified-desc order) is **empty**
  (content is empty or whitespace-only) → open that note.
- Else → create a new note and open it.
- Cursor is in the editor, ready to type.

### Dock icon click while already running
- Bring the existing window to the front. Do **not** create a new note.
- If somehow no window is open, restore/show one (still no new blank note
  unless there are zero notes).

### New note while running
- `⌘N` (unchanged) creates and focuses a new note.

## Icon

- A minimal, recognizable mark (stylized note / "nest" shape) on a dark
  rounded-square background, macOS-style, authored at 1024×1024.
- Generated dependency-free:
  - A small Swift + CoreGraphics script (committed under `scripts/`) renders
    the icon to PNG at the required sizes.
  - `iconutil` assembles an `AppIcon.icns` from a generated `.iconset`.
- `scripts/build-app.sh` copies `AppIcon.icns` into
  `NoteNest.app/Contents/Resources/` and `Info.plist` declares
  `CFBundleIconFile` (= `AppIcon`).
- Swappable: replacing the icon source (or the `.icns`) and rebuilding changes
  the icon with no code changes.

## Architecture / Changes

| File | Change |
|------|--------|
| `Sources/NoteNestKit/NotesStore.swift` | Add a helper to identify a reusable empty note. New API: `public func isEmptyContent(_ s: String) -> Bool` is overkill — instead add `public func mostRecentEmptyNote() -> Note?` returning the first note whose content is empty/whitespace-only (notes are kept newest-first), else nil. |
| `Sources/NoteNestKit/ContentView.swift` | `bootstrap()` becomes: ensure/reload; if empty list → create; else if `store.mostRecentEmptyNote()` non-nil → select it; else `store.create()`; select. |
| `Sources/NoteNest/App.swift` | App delegate gains `applicationShouldHandleReopen(_:hasVisibleWindows:)` returning `true` (default front-bring, no new note). Icon shipped via bundle (no code beyond Info.plist/Resources). |
| `scripts/make-icon.swift` (new) | CoreGraphics icon renderer → PNG sizes → `.iconset`. |
| `scripts/build-app.sh` | Build/refresh `AppIcon.icns` (via `iconutil`), copy into `Contents/Resources/`, ensure `Info.plist` has `CFBundleIconFile`. |

**Boundaries preserved:** disk access stays in `NotesStore`; the empty-note
detection is pure logic on `Note.content` and lives in `NotesStore`. View only
calls store methods.

## Testing

- **Unit (NotesStore):**
  - `mostRecentEmptyNote()` returns the newest empty/whitespace-only note when
    one exists.
  - returns `nil` when all notes have content.
  - with a mix, returns the **most recent** empty note (respects ordering).
- **Manual verification:**
  - Build via `scripts/build-app.sh`; confirm the Dock shows the new icon.
  - Launch with a non-empty recent note → opens a fresh note (or reuses the
    existing empty one); launch again without typing → no new empty file is
    created (reuse path). Verify `~/Notes` doesn't accumulate empties.
  - With the app open, click the Dock icon → existing window comes forward, no
    new note. `⌘N` still makes a new note.

## Out of Scope (YAGNI)

- Global hotkey / system-wide quick capture
- Menu bar (status item) presence
- Launch-at-login
- Icon animation, multiple themes, custom user-supplied icon UI
- Quick-switcher, search, first-line title styling (separate future work)
